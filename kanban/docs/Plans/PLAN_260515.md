# Multi-Board & Roles - Implementation Plan

## Overview

Extends the existing Kanban MVP (Parts 1-7 of the main plan complete) to support multiple boards, board-switching UI, and role-based access control across organizations.

**Starting state:** Working Next.js + ASP.NET Core + SQLite app. Single hardcoded user (`user`/`password`). One board per user. DB schema already has `boards.user_id` but no org or role concept.

**Stack additions:** EF Core migrations for schema changes; JWT claims extended with role + org.

## Part 5: Roles and Organizations

Introduce organizations and three roles (admin / manager / user) with appropriate access control.

### Schema migration
- [ ] Add EF Core migration with the following changes:
  - New table `organizations` (`id`, `name`)
  - Add nullable `org_id INTEGER REFERENCES organizations(id)` to `users`
  - Add `role TEXT NOT NULL DEFAULT 'user'` to `users` (values: `admin`, `manager`, `user`)
  - Add nullable `org_id INTEGER REFERENCES organizations(id)` to `boards`
  - New table `user_board_access` (`user_id`, `board_id`, composite PK) — records which specific boards a `user`-role user can see
- [ ] Update `docs/schema.sql` and `docs/schema.json` to reflect the new schema
